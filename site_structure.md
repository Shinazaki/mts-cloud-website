# Структура и работа сайта: MTS Cloud Portal

## 1. Архитектура проекта
Проект построен на стеке **React + Vite + TypeScript**. Для стилизации используется чистый **CSS** с мощной системой переменных, что обеспечивает высокую производительность и гибкость дизайна.

### Основные директории:
- `src/api/` — Логика взаимодействия с API. Файл `client.ts` содержит настройки axios и «заглушки» для работы без бэкенда.
- `src/contexts/` — Глобальное состояние приложения. `AuthContext` управляет сессией пользователя.
- `src/layouts/` — Общие макеты страниц:
  - `DashboardLayout` — основная панель с боковым меню и шапкой.
  - `AuthLayout` — макет страниц входа/регистрации.
- `src/pages/` — Страницы приложения (Авторизация, Серверы, Биллинг, Настройки и др.).
- `src/hooks/` — Кастомные хуки, например `useAuth` для доступа к функциям входа/выхода.

## 2. Как это работает (Основные механики)

### Авторизация и Безопасность
- **JWT Protection:** Все внутренние страницы защищены `ProtectedRoute`. Если токен отсутствует, пользователь перенаправляется на страницу входа.
- **Mock Mode:** Если ваш бэкенд не запущен, система автоматически переходит в демонстрационный режим. Вы можете войти с любыми данными, и приложение предоставит временный токен для доступа к интерфейсу.
- **Persistance:** Данные авторизации сохраняются в `localStorage`, поэтому сессия не прерывается при перезагрузке страницы.

### Управление серверами
- Страница «Сервера» отображает список виртуальных машин. 
- Реализован **живой поиск**: таблица фильтруется мгновенно при вводе имени сервера.
- Кнопка **«Создать сервер»** открывает форму выбора конфигурации (Регион, ОС, название).

### Интерфейс и Дизайн
- **Темизация:** В настройках можно переключить тему (Светлая/Темная). Это реализовано через смену классов на корневом элементе и обновление CSS-переменных.
- **Локализация:** Поддерживается переключение языков (RU/EN). Тексты интерфейса адаптируются под выбранную локаль.
- **Адаптивность:** Верстка оптимизирована под разные размеры экранов. Формы авторизации автоматически подстраиваются, чтобы всегда быть в поле зрения без прокрутки.

## 3. Сборка и запуск
- **Разработка:** `npm run dev` — запускает локальный сервер с hot-reload.
- **Сборка:** `npm run build` — создает оптимизированную версию проекта в папке `dist/`.
- **Проверка типов:** `tsc -b` — запускает проверку TypeScript кода на ошибки.
